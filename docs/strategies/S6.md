# Strategy 6 — V-Formation Liquidity Sweep Short

**Direction:** SHORT only  
**Timeframes:** 1D only  
**Config file:** `config_s6.py`  
**Evaluate function:** `evaluate_s6()` in `strategy.py`  
**Sentiment gate:** Only fires when `allowed_direction == "BEARISH"`  
**Entry mechanism:** Two-phase watcher — not immediate market entry  
**Bitget-only:** ig_bot.py does NOT use S6

---

## 1. Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `S6_ENABLED` | `True` | Master enable/disable |
| `S6_RSI_LOOKBACK` | `14` | RSI calculation period |
| `S6_SPIKE_LOOKBACK` | `30` | Max daily candles to scan for V-formation |
| `S6_OVERBOUGHT_RSI` | `70.0` | Minimum RSI at the swing-high candle |
| `S6_MIN_DROP_PCT` | `0.30` | Minimum drop from peak to spike low (30%) |
| `S6_MIN_RECOVERY_RATIO` | `0.25` | Current close must have recovered ≥25% of (peak − spike_low) range |
| `S6_SL_PCT` | `0.50` | SL = fill × (1 + 0.50) — 50% above entry (50% P/L at 10x = −500%) |
| `S6_TRAILING_TRIGGER_PCT` | `1.00` | Partial TP trigger: fill × (1 − 1.00) — 100% below entry |
| `S6_TRAIL_RANGE_PCT` | `10` | 10% trailing callback on remainder |
| `S6_LEVERAGE` | `10` | Trade leverage |
| `S6_TRADE_SIZE_PCT` | `0.04` | 4% of portfolio as margin |

## 2. When to Enter

### Phase 0 — V-Formation Pattern Detection (evaluate_s6)

`evaluate_s6()` scans the last 30 daily candles for a complete V-formation:

**Step 1 — Swing High with RSI > 70**
- Candle `i` is a swing high: `high[i] > high[i-1]` AND `high[i] > high[i+1]`
- RSI at that candle is > 70 (overbought)
- `peak_level` = that candle's high

**Step 2 — Spike Down ≥ 30%**
- After the swing high, the lowest low in subsequent candles is at least 30% below `peak_level`
- `drop_pct = (peak_level − spike_low) / peak_level ≥ 0.30`

**Step 3 — Clean Drop (no candle exceeded peak between swing high and spike)**
- No candle between the swing high and the spike low has a high above `peak_level`
- Ensures it was a clean waterfall drop, not a re-test of the high before falling

**Step 4 — Direct V-Pivot**
- The candle immediately after the spike low must be bullish (green):
  - `pivot.close > pivot.open`
  - `pivot.close > spike_candle.close`

**Step 5 — Post-Pivot Guard**
- No candle after the pivot has yet exceeded `peak_level`
- If the price already swept above the peak after the pivot, this is handled by Phase 1 in the watcher, and `evaluate_s6()` returns HOLD (the two-phase watcher handles it from that point)

**Step 6 — Recovery Ratio ≥ 25%**
- `(current_close − spike_low) / (peak_level − spike_low) ≥ 0.25`
- Rejects U-bottoms that are still consolidating near the low
- A clean V-formation will have recovered 25%+ back toward the peak

If all steps pass: returns `PENDING_SHORT` with `peak_level`, `sl_price = peak_level × 1.50`, `drop_pct`, `rsi_at_peak`.

### Phase 1 — Fakeout Sweep (two-phase watcher)

After the V-formation is detected:
- A **watcher** is registered for the symbol in `self.s6_watchers`
- Watcher polls mark price every scan tick
- **Phase 1**: wait for `mark_price > peak_level` (the liquidity sweep — price fakes out above the prior swing high to grab buy-stop liquidity)
- Once seen: `fakeout_seen = True`, `s6_fakeout_seen` in `pair_states` is set to `True`

### Phase 2 — Entry After Reversal

- **Phase 2**: after Phase 1, wait for `mark_price < peak_level` (price drops back below the peak after the fake breakout)
- When this triggers: `_execute_s6()` is called → **market SHORT entry**

### Watcher Expiry / Cancellation

- The watcher expires after **30 days** without triggering
- The watcher is cancelled if **sentiment flips BULLISH** (not safe to short in a bullish market)
- The watcher is removed if the symbol already has an open position

### Scale-in

- Initial entry is placed at **50% of target size** (`S6_TRADE_SIZE_PCT × 0.5`)
- Scale-in is queued **1 hour after entry**
- Scale-in executes the remaining 50% only if `mark_price < peak_level` (fakeout reversal still intact — price has not reclaimed the swept level)
- If price is at or above `peak_level` at the 1-hour mark, the scale-in is retried each tick until it fires or the position closes

## 3. When to Exit

### Stop Loss
- `SL = entry_fill × (1 + S6_SL_PCT / S6_LEVERAGE)` — effectively positions the SL at 50% above entry price at 10x leverage
- Note: `S6_SL_PCT = 0.50` means the SL percentage parameter is 50%, but at 10x leverage this represents a −500% P/L stop at full size — this is an extremely wide stop and functions as a catastrophic-loss guard only
- Placed as position-level SL on Bitget

### Partial TP (50%)
- `partial_trigger = fill × (1 - S6_TRAILING_TRIGGER_PCT)` = fill × 0.0 (100% below entry)
- In practice this trigger is essentially never reached as a fixed price target (price cannot go to 0)
- The intent is that the trailing stop activates first

### Trailing Stop on Remaining 50%
- Exchange-side `moving_plan` with 10% callback (`S6_TRAIL_RANGE_PCT`)
- Activated at the same 100% trigger level
- In practice, the trailing stop is the primary exit mechanism for S6

## 4. state.json Effects

Fields updated in `pair_states[symbol]`:

```python
{
  "s6_signal":       str,    # "PENDING_SHORT" | "HOLD"
  "s6_reason":       str,    # reason string (V-formation details)
  "s6_peak_level":   float,  # high of swing-high candle (liquidity level to sweep)
  "s6_sl":           float,  # SL = peak_level × 1.50
  "s6_fakeout_seen": bool,   # True after Phase 1 fakeout (mark > peak_level) observed
}
```

`s6_fakeout_seen` is reset to `False` each scan cycle in `update_pair_state()`, then patched to `True` by `_process_s6_watchers()` when Phase 1 is observed during the same tick.

## 6. Active Trade Behavior (per tick)

While an S6 trade is open:

1. **PnL sync**: `unrealised_pnl` updated.
2. **Partial TP detection**: if live qty drops below 75% of initial (would indicate the `profit_plan` triggered at −100%), `S6_PARTIAL` logged.
3. **No swing trail**: S6 has no bot-side structural trailing (uses exchange-side `moving_plan` only).
4. **Close detection**: when position disappears, `S6_CLOSE` logged.

On close, `S6_CLOSE` appended to `trades.csv`. Pair pause rule applies: 3 losses in one UTC day → pair paused until midnight.

## 9. Two-Phase Logic Summary

```
evaluate_s6() returns PENDING_SHORT
        ↓
_queue_s6_watcher() registered
        ↓
Phase 1: mark > peak_level  →  fakeout_seen = True
        ↓
Phase 2: mark < peak_level  →  _execute_s6() fires SHORT
```

The key insight: the V-formation identifies a prior peak with overbought RSI that had a violent drop. The "fakeout sweep" is when price recovers above that peak to grab buy-stop orders (liquidity), then reverses. The SHORT entry captures that reversal.
