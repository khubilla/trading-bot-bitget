# Strategy 5 — SMC Order Block Pullback

**Direction:** LONG (bullish bias) and SHORT (bearish bias)  
**Timeframes:** 1D (EMA bias), 1H (Break of Structure), 15m (Order Block + entry)  
**Config file:** `config_s5.py` (Bitget) / per-instrument CONFIG dict (IG bot)  
**Evaluate function:** `evaluate_s5()` in `strategy.py`  
**Entry mechanism:** PENDING limit order — not immediate market entry

---

## 2. When to Enter

### Signal generation — all conditions must pass:

**Step 0 — Sentiment Gate**
- `allowed_direction` must be `"BULLISH"` (to consider LONG) or `"BEARISH"` (to consider SHORT)
- If `"NEUTRAL"`, returns HOLD immediately — neither branch runs

**Step 1 — Daily EMA Bias**
- LONG: EMA10 > EMA20 > EMA50 (bullish daily stack)
- SHORT: EMA50 > EMA20 > EMA10 (bearish daily stack)

**Step 2 — 1H Break of Structure (BOS)**

BOS is treated as a **structural price level**, not a per-candle comparison. Once price breaks above a swing high (LONG) or below a swing low (SHORT), that level remains valid as long as price stays on the correct side.

Implementation: scan backwards through the last `S5_HTF_BOS_LOOKBACK` completed 1H candles (using `iloc[-1]` as the current live price — safe because the BOS *level* is derived from completed candles only via `:-1` window slice). Find the **most recently broken** pivot:

- LONG (`bos_high`): the most recent 1H pivot high (a candle whose high is strictly greater than both its neighbours) where the current 1H close is **above** that pivot's high → BOS confirmed; price has broken a structural high and remains above it
- SHORT (`bos_low`): the most recent 1H pivot low (a candle whose low is strictly less than both its neighbours) where the current 1H close is **below** that pivot's low → BOS confirmed

If no such broken pivot is found in the lookback window, BOS is **not** confirmed → `HOLD`.

**Step 3 — 15m Order Block Detection**

*For LONG (Bullish OB):*
- Scan the last 50 15m candles for the most recent: ≥2 consecutive green candles moving price up ≥1% (bullish impulse)
- The OB = the last **red** (bearish) candle immediately before that impulse run
- `ob_low` = red candle's low (outer SL boundary)
- `ob_high` = red candle's open (top of body = entry level)
- OB range (`(ob_high - ob_low) / ob_low`) must be ≥ 0.5% (rejects flat candles)

*For SHORT (Bearish OB):*
- Scan for ≥2 consecutive red candles moving price down ≥1% (bearish impulse)
- The OB = the last **green** (bullish) candle immediately before the impulse
- `ob_low` = green candle's open (bottom of body = entry level)
- `ob_high` = green candle's high (outer SL boundary)

**Step 4 — OB Touch**
- Any candle within the last `S5_CHOCH_LOOKBACK` 15m candles has a low that dipped into or through `ob_high × 1.002` (LONG), confirming the pullback has reached the OB zone

**Step 5 — Stale OB Guard**
- LONG: if current 15m close is already > `ob_high × 1.04`, skip (OB is stale — price left too far above)
- SHORT: if current close is already < `ob_low × 0.96`, skip

**Step 6 — Structural TP**
- LONG: `find_swing_high_target()` finds the nearest 15m swing high above `ob_high`
- SHORT: `find_swing_low_target()` finds the nearest 15m swing low below `ob_low`
- If no swing target found, HOLD (no trade)

**Step 7 — R:R ≥ 2.0**
- LONG: R:R = (tp_price − ob_high) / (ob_high − sl_price)
- SHORT: R:R = (ob_low − tp_price) / (sl_price − ob_low)
- Must be ≥ `S5_MIN_RR`

**Optional: FVG Filter (if `S5_SMC_FVG_FILTER = True`)**
- Must find an unfilled bullish/bearish FVG in the last 20 15m candles near the OB

### Signal returned: `PENDING_LONG` or `PENDING_SHORT`

The evaluate function returns a pending signal, not a market entry. The actual entry is a **limit order** placed on Bitget at the OB boundary price.

### How the pending signal becomes a trade

1. `evaluate_s5()` returns `PENDING_LONG` / `PENDING_SHORT` with `entry_trigger = ob_high` (LONG) or `ob_low` (SHORT)
2. `_queue_s5_pending()` pre-validates (Claude filter) and adds the signal to `self.pending_signals[symbol]`
3. The **entry watcher thread** (polls every 4s) monitors for:
   - Order fill status via Bitget API
   - OB invalidation: checks if last completed 15m candle closed through the OB boundary
   - Expiry timeout
4. When limit order fills, `_handle_limit_filled()` registers the trade and places exit orders
5. The pending signal is invalidated if the OB is broken (15m candle close confirmation):
   - LONG: last completed 15m candle closes below `ob_low × (1 - 0.1%)`
   - SHORT: last completed 15m candle closes above `ob_high × (1 + 0.1%)`
   - **Note:** Uses candle close, not instant mark price — filters out wicks and noise while staying responsive (max 15 min delay)

### Scale-in
S5 does **not** use scale-in. Full size at entry.

## 3. When to Exit

### Stop Loss
- LONG: `SL = ob_low × (1 - 0.3%)`
- SHORT: `SL = ob_high × (1 + 0.3%)`
- Placed as position-level SL on Bitget

### Partial TP (50%) at 1:1 R:R
- `partial_trigger = entry + risk` (LONG) or `entry - risk` (SHORT) where `risk = |entry - sl|`
- 50% of position closed at this 1:1 level
- Placed as `profit_plan` order at entry time

### Structural TP (remaining 50%)
- If `tp_price > 0`: hard TP at the swing target level, placed as fixed `profit_plan` for `rest_qty`
- Placed as `place-tpsl-order profit_plan` at `tp_price`

### Fallback Trailing Stop (if no structural TP)
- If no swing target was found, `moving_plan` with 5% callback (`S5_TRAIL_RANGE_PCT`) placed for rest_qty

### Bot-Side Swing Trail (`S5_USE_SWING_TRAIL = True`, Reference-High/Low Gated)

Same reference-gated cycle as S3, on 15m candles, runs from trade open.

**LONG:**
- Init: `swing_trail_ref` = nearest 15m swing high above current price
- Each tick: if `mark_price >= ref_high` → find swing low formed after the ref candle → `SL = swing_low × (1 − S5_SL_BUFFER_PCT)`; only apply if higher than current SL → reset ref to next 15m swing high above current price

**SHORT (mirror):**
- Init: `swing_trail_ref` = nearest 15m swing low below current price
- Each tick: if `mark_price <= ref_low` → find swing high formed after the ref candle → `SL = swing_high × (1 + S5_SL_BUFFER_PCT)`; only apply if lower than current SL → reset ref to next 15m swing low below current price

`ap["swing_trail_ref"]` is in-memory only — not persisted to state.json or CSV.

## 4. state.json Effects

Fields updated in `pair_states[symbol]`:

```python
{
  "s5_signal":        str,    # "PENDING" (displayed) | "LONG" | "SHORT" | "HOLD"
  "s5_reason":        str,    # descriptive reason string
  "s5_ob_low":        float,  # Order Block lower boundary
  "s5_ob_high":       float,  # Order Block upper boundary
  "s5_entry_trigger": float,  # ob_high (LONG) or ob_low (SHORT)
  "s5_sl":            float,  # stop loss price
  "s5_tp":            float,  # structural TP price
  "s5_sr_pct":        float,  # S/R clearance % (handled internally via R:R)
  "s5_priority_rank":  int,   # rank among all candidates this cycle (1=best)
  "s5_priority_score": float, # score used for ranking
}
```

Note: `s5_signal` is stored as `"PENDING"` in pair_states when the signal is `PENDING_LONG` or `PENDING_SHORT` (the raw PENDING_LONG/SHORT is used internally by the entry watcher).

## 6. Active Trade Behavior (per tick)

While an S5 trade is open:

1. **PnL sync**: `unrealised_pnl` updated.
2. **Partial TP detection**: if live qty drops below 75% of initial, `S5_PARTIAL` logged.
3. **Candle stop trail** (`S5_USE_CANDLE_STOPS = True`): SL stepped to nearest 15m swing low/high, favourably only.
4. **Close detection**: when position disappears, `S5_CLOSE` logged.

There is no scale-in for S5.

## 8. Trade History on Exit

On close, `S5_CLOSE` appended to `trades.csv`. Pair pause rule applies: 3 losses in one UTC day → pair paused until midnight.
