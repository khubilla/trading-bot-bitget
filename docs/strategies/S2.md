# Strategy 2 — Daily Momentum Coil Breakout

**Direction:** LONG only  
**Timeframes:** 1D only  
**Config file:** `config_s2.py`  
**Evaluate function:** `evaluate_s2()` in `strategy.py`

---

## 1. Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `S2_ENABLED` | `True` | Master enable/disable |
| `S2_BIG_CANDLE_BODY_PCT` | `0.20` | Minimum 20% body size to qualify as momentum candle |
| `S2_BIG_CANDLE_LOOKBACK` | `30` | Search last 30 daily candles for the big candle |
| `S2_RSI_LONG_THRESH` | `70` | Daily RSI must be above this |
| `S2_CONSOL_CANDLES` | `5` | Max 5 tight consolidation candles (tries 1 to 5, picks smallest valid) |
| `S2_CONSOL_RANGE_PCT` | `0.15` | Max 15% effective range for tight consolidation |
| `S2_BREAKOUT_BUFFER` | `0.01` | 1% buffer above box top for entry trigger |
| `S2_DARVAS_WICK_PCT` | `0.05` | Wick > 5% of body top → use body; ≤ 5% → use wick |

| `S2_LEVERAGE` | `10` | Trade leverage |
| `S2_TRADE_SIZE_PCT` | `0.04` | 4% of portfolio as margin (initial entry uses 50% = 2%) |

| `S2_STOP_LOSS_PCT` | `0.05` | SL cap: 5% price move from entry (box_low is primary SL) |
| `S2_TRAILING_TRIGGER_PCT` | `0.10` | Trail trigger: +10% from entry |
| `S2_TRAILING_RANGE_PCT` | `10` | 10% trailing callback on remaining 50% |
| `S2_USE_SWING_TRAIL` | `False` | S2 uses exchange-side % trailing stop (swing trail off) |
| `S2_SWING_LOOKBACK` | `30` | Daily candles for swing low search (if swing trail enabled) |
| `S2_MIN_SR_CLEARANCE` | `0.15` | Skip if resistance above spike peak is < 15% from entry |

## 2. When to Enter

### All conditions must pass in order (daily chart only):

**Step 1 — Big Momentum Candle**
- At least one candle in the last 30 daily candles has a body ≥ 20% of its open price
- Candle close must be above prior consolidation range (implied by the coil logic below)

**Step 2 — Daily RSI > 70**
- Current daily RSI (14-period) must be above 70

**Step 3 — Tight Daily Coil (1–5 candles)**
- The most recent 1 to 5 completed daily candles form a tight consolidation:
  - Floor = `eff_high × (1 − S2_CONSOL_RANGE_PCT)` — effective lows must not fall below this (equivalently: `(eff_high − eff_low) / eff_high ≤ S2_CONSOL_RANGE_PCT`)
  - Effective high and low are computed using the Darvas Box body/wick rule (see GENERAL_CONCEPTS.md §1)
  - All consolidation candles must close at or below the big candle's body top
  - All consolidation candles must have daily RSI > 70 throughout
- The algorithm tries the smallest valid window first (1 candle, then 2, up to 5)

**Step 4 — Breakout Above the Entry Trigger**
- The current daily close exceeds the entry trigger
- Entry trigger = box_high + 1% buffer (using Darvas Box Top Rule for wick/body selection)
- Entry trigger is floored against the big candle's body top
- If price is already > 4% above the trigger at execution time, the entry is skipped (missed move)

**S/R Clearance Gate** (applied at execution, not in `evaluate_s2()`):
- Find the highest high in the last 30 daily candles (the spike peak)
- Search for the nearest resistance **above** that spike peak
- If resistance is < 15% above current price, skip the trade

### Scale-in

- Initial entry is placed at **50% of target size** (`S2_TRADE_SIZE_PCT × 0.5`)
- Scale-in is queued **1 hour after entry**
- Scale-in executes the remaining 50% only if price is between `box_high` and `box_high × 1.04`
- If price is outside the window at the 1-hour mark, the scale-in is retried each tick until it fires or the position closes

---

## 3. When to Exit

### Stop Loss
- Initial SL = `max(box_low × 0.999, entry × (1 - 5%))`
- `box_low` is the bottom of the daily consolidation coil. When the entry trigger is floored to the big candle's body top (see Step 4), `box_low` is set to `entry_trigger × (1 − S2_CONSOL_RANGE_PCT)` since the raw consolidation sits below the relevant level.
- The 5% cap prevents the box_low SL from being catastrophically far below entry
- Placed as a position-level SL on Bitget (`place-pos-tpsl`) — scales automatically with scale-in

### Partial TP (50%)
- 50% of position closed at entry + 10% (`S2_TRAILING_TRIGGER_PCT`)
- Placed as a `profit_plan` order on Bitget at entry time

### Trailing Stop on Remaining 50%
- Exchange-side `moving_plan` with 10% callback (`S2_TRAILING_RANGE_PCT`)
- Triggered at same +10% level as partial TP

### After Scale-in
- `profit_plan` and `moving_plan` orders are cancelled and re-placed with new total qty
- New trail trigger = `new_avg_entry × 1.10`
- SL cap is recalculated from new average entry: `max(box_low × 0.999, new_avg × 0.95)`; raised only if higher

### Bot-Side Swing Trail (post-partial, if `S2_USE_SWING_TRAIL = True`, Reference-High Gated)

Same reference-gated cycle as S3, on daily candles, activates only after partial TP has fired.

- Init: `swing_trail_ref` = nearest daily swing high above current price
- Each tick (post-partial): if `mark_price >= ref_high` → find swing low formed after the ref candle → `SL = swing_low × (1 − S2_STOP_LOSS_PCT)`; only apply if higher than current SL → reset ref to next daily swing high above current price

SL never moves before partial TP fires and never moves against the trade.

## 6. Active Trade Behavior (per tick)

While an S2 trade is open:

1. **PnL sync**: `unrealised_pnl` updated from exchange.
2. **Scale-in check**: if `scale_in_pending=True` and `time.time() >= scale_in_after`, `_do_scale_in()` is called.
3. **Partial TP detection**: if live position qty dropped below 75% of initial qty, `S2_PARTIAL` is logged.
4. **Swing trail** (if enabled and partial fired): SL stepped to nearest daily swing low.
5. **Close detection**: when position disappears, `S2_CLOSE` logged and trade removed.

## 7. Pair Scanner Display

Dashboard fields for S2:

- `s2_signal` — current signal
- `s2_reason` — reason string (includes coil count, big candle %, RSI, trigger details)
- `s2_daily_rsi` — daily RSI
- `s2_big_candle` — whether a qualifying big candle was found
- `s2_coiling` — whether a valid coil is detected and waiting breakout
- `s2_box_low` / `s2_box_high` — coil boundaries (when trigger is floored to big candle body top, these reflect the floored trigger range: `box_high = entry_trigger`, `box_low = entry_trigger × (1 − S2_CONSOL_RANGE_PCT)`)
- `s2_sr_resistance_pct` / `s2_sr_resistance_price` — resistance clearance

## 8. Trade History on Exit

On close, `S2_CLOSE` appended to `trades.csv`. Win/loss is determined by the close-portion PnL. Partial PnL is already separately logged and subtracted to avoid double-counting.

Pair pause rule applies: 3 losses in one UTC day → pair paused until midnight.
