# Security Regression Testing Design

**Date:** 2026-03-31
**Status:** Approved

## Summary

A two-file pytest security regression suite that runs after existing unit tests. Uses a real uvicorn server (spun up autonomously via fixture) for application-level tests, plus subprocess-based dependency and static analysis scanners. Tests assert desired secure behavior — they fail now and pass once vulnerabilities are fixed.

---

## Scope

### What is being tested

| Layer | File | Tool | Catches |
|-------|------|------|---------|
| Application | `tests/test_security.py` | httpx + real uvicorn | Auth, rate-limit, input, CORS, info-exposure, injection, security headers |
| Dependencies | `tests/test_security_scan.py` | pip-audit | New CVEs in installed packages as they are disclosed |
| Source code | `tests/test_security_scan.py` | bandit | Insecure patterns introduced in new code |

### What is NOT in scope

- Penetration testing or active exploitation
- Network-level port scanning (0.0.0.0 binding is a deployment config issue, not tested here)
- Authentication implementation (tests define what auth must do; implementation is a separate task)
- IG bot dashboard (only Bitget dashboard tested; IG follows same pattern)

---

## Architecture

### Server Fixture (`tests/conftest.py` — extend existing)

A **session-scoped** pytest fixture `live_server_url` that:

1. Patches `state.load_state()` to return mock data — no real `.env`, Bitget, IG, or Anthropic credentials required
2. Imports and creates the FastAPI `app` object from `dashboard.py`
3. Starts uvicorn in a `threading.Thread` on port **8099** (avoids conflict with live 8080 and paper 8081)
4. Polls `GET /api/state` with a 5-second timeout until the server responds
5. Yields `"http://127.0.0.1:8099"` to all tests
6. Signals uvicorn to shut down and joins the thread after the session

The fixture uses `uvicorn.Config` + `uvicorn.Server` directly (not subprocess) so it shares the same Python process and can receive mock patches.

No real credentials are needed because all external API calls (Bitget, IG, Anthropic) are patched at the module level before the app starts.

### Test Files

```
tests/
  conftest.py              # extend with live_server_url fixture
  test_security.py         # application-level tests (requires live_server_url)
  test_security_scan.py    # pip-audit + bandit (no server needed)
```

## `test_security.py` — Application Tests

### Dependencies

- `httpx` — HTTP client (add to `requirements.txt`)

### Test Classes

#### `TestAuthentication`

Asserts all dashboard endpoints return **401 Unauthorized** when no auth token is provided.

Endpoints covered:
- `GET /`
- `GET /api/state`
- `POST /api/chat`
- `GET /api/candles/BTCUSDT`
- `GET /api/entry-chart`
- `GET /api/trade-chart`
- `GET /api/ig/state`

Expected fix: a simple static API token in an `Authorization: Bearer <token>` header, configured via `.env` (`DASHBOARD_API_KEY`). Token is generated by the user and stored in `.env`.

#### `TestRateLimiting`

Asserts `POST /api/chat` returns **429 Too Many Requests** after exceeding a threshold (e.g., 10 requests per 60 seconds per IP).

Tests:
- 10 rapid requests → all succeed (within limit)
- 11th request → 429

Note: the "rate limit resets after window" case is not tested — it would require sleeping 60s. The 429 assertion is sufficient for regression purposes.

Expected fix: `slowapi` or a simple in-memory counter middleware on the `/api/chat` route only (not all routes — trading data endpoints need low latency).

#### `TestInputValidation`

Asserts `GET /api/candles/{symbol}` rejects invalid symbol values with **400 Bad Request**.

Invalid inputs tested:
- Path traversal: `../etc/passwd`, `..%2Fetc%2Fpasswd`
- Oversized: string longer than 20 characters
- Invalid characters: spaces, semicolons, slashes, null bytes
- Empty string

Valid inputs that must pass (allowlist):
- `BTCUSDT`, `ETHUSDT`, `SOLUSDT` — uppercase alphanumeric + max 20 chars

Expected fix: a regex allowlist validator on the `symbol` path parameter in `dashboard.py` (`^[A-Z0-9]{2,20}$`).

#### `TestCORS`

Asserts that cross-origin requests from non-localhost origins are rejected (no `Access-Control-Allow-Origin: *` header).

Tests:
- Request with `Origin: https://evil.example.com` → no ACAO header or ACAO is not `*`
- Request with `Origin: http://localhost:8099` → ACAO header allows localhost
- Request with `Origin: http://127.0.0.1:8099` → ACAO header allows 127.0.0.1

Expected fix: add FastAPI `CORSMiddleware` with `allow_origins=["http://localhost:8080", "http://127.0.0.1:8080"]`.

#### `TestInformationExposure`

Asserts that error responses do not leak internal details.

Tests:
- `GET /api/candles/INVALIDSYMBOL` → response body does not contain file paths (e.g., `/Users/`, `/home/`)
- `GET /nonexistent` → 404 body does not contain Python traceback keywords (`Traceback`, `File "`, `line `)
- Any 500 response → body does not contain stack trace

Expected fix: add a global exception handler in `dashboard.py` that returns a generic `{"error": "internal server error"}` for unhandled exceptions.

#### `TestAPIProxyAbuse`

Asserts that `/api/chat` rejects oversized payloads to prevent Anthropic API abuse.

Tests:
- `messages` field > 10,000 characters → **413 Request Entity Too Large**
- `trade` field > 5,000 characters → **413**
- Normal-sized payload → passes through

Expected fix: add a payload size check at the start of the `/api/chat` handler before calling `claude_analyst.stream_response`.

#### `TestInjectionAttacks`

Asserts that injection payloads sent to `/api/candles/{symbol}` and `/api/chat` are rejected before reaching any processing layer.

**Command/OS injection payloads** in `symbol` param — all must return **400**:
- `$(python -c 'import os; os.getcwd()')`
- `` `id` ``
- `BTCUSDT; python -c 'import socket'`
- `BTCUSDT | python -c 'import sys'`
- `BTCUSDT && whoami`

**SQL-style injection payloads** in `symbol` param — all must return **400**:
- `' OR 1=1--`
- `'; DROP TABLE orders;--`
- `1 UNION SELECT NULL--`

**Header injection** via newlines in `symbol` param — all must return **400**:
- `BTCUSDT\r\nX-Injected: evil`
- `BTCUSDT\nSet-Cookie: session=hijacked`

**JSON structure attacks** in `/api/chat` body — all must return **400** or be safely ignored:
- Deeply nested object (`{"a": {"a": {"a": ...}}}` 100 levels deep)
- Array where string expected (`"messages": [[[["payload"]]]`)
- Null bytes in string fields (`"trade": "data\x00injection"`)

Note: these payloads will not actually execute code — FastAPI does not shell-execute path params. The tests confirm the allowlist and input validation layers are enforced, protecting against future refactors that might introduce execution paths.

Expected fix: covered by the symbol allowlist regex (`^[A-Z0-9]{2,20}$`) from `TestInputValidation` and payload validation from `TestAPIProxyAbuse`. `bandit` static analysis in `test_security_scan.py` independently confirms no `subprocess`/`eval`/`exec` calls use user-controlled input in the source.

#### `TestSecurityHeaders`

Asserts that every dashboard response includes the required HTTP security headers.

Endpoints checked: `GET /`, `GET /api/state`, `GET /api/candles/BTCUSDT`

Required headers and values:
- `X-Frame-Options: DENY` — prevents clickjacking the dashboard in an iframe
- `X-Content-Type-Options: nosniff` — prevents MIME-type sniffing attacks
- `Content-Security-Policy` — present, does not contain `unsafe-inline` for `script-src` (TradingView CDN must be explicitly listed instead)
- `Referrer-Policy: no-referrer` — suppresses referrer leakage on outbound links
- `Server` header — must be absent or not expose `uvicorn` version string

Expected fix: add a response middleware in `dashboard.py` that injects these headers on every response. Example:

```python
@app.middleware("http")
async def add_security_headers(request, call_next):
    response = await call_next(request)
    response.headers["X-Frame-Options"] = "DENY"
    response.headers["X-Content-Type-Options"] = "nosniff"
    response.headers["Referrer-Policy"] = "no-referrer"
    response.headers["Content-Security-Policy"] = (
        "default-src 'self'; "
        "script-src 'self' https://unpkg.com; "
        "style-src 'self' 'unsafe-inline'"
    )
    response.headers.pop("server", None)
    return response
```

## `test_security_scan.py` — Scanner Tests

### Dependencies

- `pip-audit` — install as dev dependency (`pip install pip-audit`)
- `bandit` — install as dev dependency (`pip install bandit`)

Neither is added to `requirements.txt` (they are dev/CI tools only). Track them in a new `requirements-dev.txt` alongside `httpx` and `pytest`.

### Test Functions

#### `test_no_dependency_cves`

Runs `pip-audit --format json` as a subprocess (outputs JSON to stdout by default). Parses JSON output. Fails the test if any vulnerability is found, printing:

```
FAIL: 3 CVE(s) found in dependencies:
  requests 2.28.0 — CVE-2023-32681 (fix: upgrade to 2.31.0)
  ...
```

Passes silently if no CVEs found.

#### `test_no_high_severity_code_issues`

Runs `bandit -r . -ll --format json --exclude ./tests,./docs` as a subprocess (medium + high severity only; `-ll` skips low-noise low-severity findings). Parses JSON output. Fails if any issues found, printing file, line number, issue description, and severity.

Skips `tests/` and `docs/` directories to avoid false positives from test code.

## Integration with Existing Test Suite

Run order (in CI or locally):

```bash
pytest tests/                          # all tests including security
pytest tests/test_security.py          # application security only
pytest tests/test_security_scan.py     # scanners only
```

Security tests are **not** marked with any special marker — they run as part of the default suite. The live server fixture adds ~2–3 seconds startup overhead once per session.

---

## Known Failing Tests at Merge Time

All tests in `test_security.py` are expected to fail until the following fixes are implemented (separate tasks, not part of this spec):

1. Add `DASHBOARD_API_KEY` bearer token auth middleware to all dashboard routes
2. Add `slowapi` rate limiting to `/api/chat`
3. Add symbol regex allowlist to `/api/candles/{symbol}`
4. Add `CORSMiddleware` restricted to localhost origins
5. Add global exception handler suppressing stack traces
6. Add payload size validation to `/api/chat` handler
7. Add security headers response middleware to `dashboard.py`

Scanner tests in `test_security_scan.py` may pass immediately if no current CVEs or bandit issues exist.
