# Strategy 4 — Post-Pump RSI Divergence Short

**Direction:** SHORT only  
**Timeframes:** 1D only  
**Config file:** `config_s4.py`  
**Evaluate function:** `evaluate_s4()` in `strategy.py`  
**Sentiment gate:** Only fires when sentiment is NOT BULLISH (BEARISH or NEUTRAL)

---

## 1. Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `S4_ENABLED` | `True` | Master enable/disable |
| `S4_BIG_CANDLE_BODY_PCT` | `0.20` | Min 20% body to qualify as spike candle |
| `S4_BIG_CANDLE_LOOKBACK` | `30` | Search last 30 daily candles for spike |
| `S4_RSI_PEAK_THRESH` | `75` | RSI must have reached this level (overbought) |
| `S4_RSI_PEAK_LOOKBACK` | `10` | Last N daily candles to search for RSI peak |
| `S4_RSI_DIV_MIN_DROP` | `5` | RSI bearish divergence: 2nd push ≥5 pts lower than 1st |
| `S4_RSI_STILL_HOT_THRESH` | `70` | Previous candle RSI must still be ≥70 — else setup invalid |
| `S4_ENTRY_BUFFER` | `0.01` | Entry = prev_low × (1 - 1%) |
| `S4_MAX_ENTRY_BUFFER` | `0.04` | Skip if price is already > 4% below prev_low |
| `S4_TRAILING_TRIGGER_PCT` | `0.10` | Partial TP: price −10% from entry |
| `S4_TRAILING_RANGE_PCT` | `10` | 10% trailing callback on remaining 50% |
| `S4_USE_SWING_TRAIL` | `False` | S4 uses exchange-side % trailing stop only |
| `S4_SWING_LOOKBACK` | `30` | Daily candles for swing high search (if swing trail enabled) |
| `S4_LEVERAGE` | `10` | Trade leverage |
| `S4_TRADE_SIZE_PCT` | `0.04` | 4% of portfolio as margin (initial uses 50% = 2%) |
| `S4_MIN_SR_CLEARANCE` | `0.15` | Skip if spike base (pre-pump support) < 15% below entry |
| `S4_LOW_LOOKBACK` | `5` | 1H candle low filter: entry trigger must be ≤ lowest low of last N 1H candles |

## 2. When to Enter

### All conditions must pass (daily chart only):

**Step 1 — Big Spike Candle**
- At least one daily candle in the last 30 has a body ≥ 20% of open price (bullish spike)

**Step 2 — RSI Was Overbought**
- Within the last 10 daily candles, RSI (14-period) peaked above 75 (`S4_RSI_PEAK_THRESH`)

**Step 2b — RSI Still Hot**
- The **previous daily candle** (iloc[-2]) RSI must be ≥ 70 (`S4_RSI_STILL_HOT_THRESH`)
- If RSI has already cooled below 70 on the prev candle, the setup is considered invalidated (pump already faded)

**Step 3 — Optional RSI Bearish Divergence**
- Split the RSI lookback window in half; if the first half's peak was higher than the second half's peak by ≥ 5 points → divergence detected
- This is informational / boosting, not a hard gate; the trade still fires without divergence

**Step 4 — Entry Trigger: Previous Day's Low**
- `entry_trigger = prev_daily_low × (1 - 1%)`
- Trade fires when the current price breaches this level (intraday break of prior day's low)
- If price is already > 4% below prev_daily_low, the entry is skipped (missed move)

**Step 5 — 1H Low Filter**
- Look at the last `S4_LOW_LOOKBACK` (default 5) completed 1H candles
- Find the lowest low among them: `min_htf_low`
- Skip if `entry_trigger > min_htf_low` (entry is not breaking the 1H structure low)
- This ensures the daily breakdown aligns with a multi-hour structural low

**S/R Clearance Gate** (applied at execution):
- `find_spike_base()` locates the high of the most recent bullish spike candle below current price (the pre-pump base = breakout level that is now potential support)
- If that base is < 15% below current entry price, the trade is skipped (price is too close to support)

### Scale-in

- Initial entry is placed at **50% of target size** (`S4_TRADE_SIZE_PCT × 0.5`)
- Scale-in queued **1 hour after entry**
- Scale-in executes the remaining 50% only if **both** conditions hold:
  1. Market sentiment is **BEARISH** (not NEUTRAL) — scale-in is skipped while sentiment is NEUTRAL and retried each tick
  2. Price is between `prev_low × (1 - 4%)` and `prev_low × (1 - 1%)`

---

## 3. When to Exit

### Stop Loss
- `SL = entry_price × (1 + 50% / leverage)` = effectively a −50% P/L stop at 10x
- Placed as position-level SL on Bitget

### Partial TP (50%)
- 50% closed when price drops 10% from entry (`S4_TRAILING_TRIGGER_PCT`)
- Placed as `profit_plan` order at entry time

### Trailing Stop on Remaining 50%
- Exchange-side `moving_plan` with 10% callback
- Activated at the same −10% trigger level

### After Scale-in
- `profit_plan` + `moving_plan` cancelled and re-placed with new total qty
- New trail trigger = `new_avg_entry × (1 - 10%)`
- SL remains structural (spike high × buffer) — no SL update after scale-in for S4

### Bot-Side Swing Trail (post-partial, if `S4_USE_SWING_TRAIL = True`, Reference-Low Gated)

Mirror of S2's logic for SHORT, on daily candles, activates only after partial TP has fired.

- Init: `swing_trail_ref` = nearest daily swing low below current price
- Each tick (post-partial): if `mark_price <= ref_low` → find swing high formed after the ref candle → `SL = swing_high × (1 + S4_ENTRY_BUFFER)`; only apply if lower than current SL → reset ref to next daily swing low below current price

SL never moves before partial TP fires and never moves against the SHORT trade.

|--------|-------|

On **partial TP** (`S4_PARTIAL`): `pnl`, `pnl_pct`, `exit_reason="PARTIAL_TP"`, `exit_price`  
On **close** (`S4_CLOSE`): `result`, `pnl`, `pnl_pct`, `exit_reason`, `exit_price`

---

## 6. Active Trade Behavior (per tick)

While an S4 trade is open:

1. **PnL sync**: `unrealised_pnl` updated.
2. **Scale-in check**: if `scale_in_pending=True` and 1 hour elapsed, `_do_scale_in()` called.
3. **Partial TP detection**: if live qty drops below 75% of initial, `S4_PARTIAL` logged.
4. **Swing trail** (if enabled and partial fired): SL stepped to nearest daily swing high, down only.
5. **Close detection**: when position disappears, `S4_CLOSE` logged.
