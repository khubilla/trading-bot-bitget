**Direction:** LONG and SHORT  
**Timeframes:** 1D (trend), 1H (HTF break), 3m (entry)  
**Config file:** `config_s1.py`  
**Evaluate function:** `evaluate_s1()` in `strategy.py`

## 2. When to Enter

### All conditions must pass in order:

**Step 1 ŌĆö 1H HTF Structure Break**
- LONG: current 1H high > previous 1H high (higher high = bullish break)
- SHORT: current 1H low < previous 1H low (lower low = bearish break)

**Step 2 ŌĆö Daily Trend Filter**
- Daily ADX > 25 (trending, not ranging)
- LONG: daily close > EMA20 (uptrend context)
- SHORT: daily close < EMA20 (downtrend context)
- Also applies the S/R clearance gate: nearest daily resistance (LONG) or support (SHORT) must be Ōēź 15% away from current price.

**Step 3 ŌĆö 3m RSI Zone**
- LONG: current 3m RSI > 70
- SHORT: current 3m RSI < 30

**Step 4 ŌĆö 3m Coil (RSI-zone aware)**
- Floor = `box_high ├Ś (1 ŌłÆ CONSOLIDATION_RANGE_PCT)` ŌĆö effective lows must not fall below this (equivalently: `(box_high ŌłÆ box_low) / box_high Ōēż CONSOLIDATION_RANGE_PCT`, i.e. Ōēż 0.3%)
- RSI must have been in the zone (>70 LONG, <30 SHORT) throughout those 2 candles

**Step 5 ŌĆö 3m Breakout**
- LONG: current 3m close > box_high ├Ś (1 + 0.5%)
- SHORT: current 3m close < box_low ├Ś (1 - 0.5%)

### Scale-in
S1 does **not** use scale-in. Full size is entered at the breakout candle.

## 3. When to Exit

### Stop Loss
- Initial SL = `max(entry ├Ś (1 - 5%), box_low ├Ś (1 - 0.5%))` for LONG
- Initial SL = `min(entry ├Ś (1 + 5%), box_high ├Ś (1 + 0.5%))` for SHORT
- This is placed as a position-level SL on Bitget (`place-pos-tpsl`)

### Partial TP (50%)
- 50% of position closed at entry ┬▒ 10% (`TAKE_PROFIT_PCT`)
- Placed as `profit_plan` order on Bitget at entry time

### Trailing Stop on Remaining 50%
- Exchange-side `moving_plan` with 5% callback (`S1_TRAIL_RANGE_PCT`)
- Activated at the same trigger level as the partial TP (+10% for LONG, ŌłÆ10% for SHORT)

### Bot-Side Swing Trail (Reference-High/Low Gated)

Uses the same reference-gated cycle as S3, adapted for both LONG and SHORT on 3m candles.

**LONG:**
- Init: `swing_trail_ref` = nearest 3m swing high above current price
- Each tick: if `mark_price >= ref_high` ŌåÆ find swing low formed after the ref candle ŌåÆ `SL = swing_low ├Ś (1 ŌłÆ S1_SL_BUFFER_PCT)`; only apply if higher than current SL ŌåÆ reset ref to next swing high above current price

**SHORT (mirror):**
- Init: `swing_trail_ref` = nearest 3m swing low below current price
- Each tick: if `mark_price <= ref_low` ŌåÆ find swing high formed after the ref candle ŌåÆ `SL = swing_high ├Ś (1 + S1_SL_BUFFER_PCT)`; only apply if lower than current SL ŌåÆ reset ref to next swing low below current price

**Key rule:** SL does not move when the reference level is broken. It waits for a new swing low/high to form **after** that break. That level becomes the new SL and the cycle repeats.

`ap["swing_trail_ref"]` is in-memory only ŌĆö not persisted to state.json or CSV.

### Advisory Box Break
- Each tick the bot checks if the last closed 3m candle broke the box boundary in the losing direction.
- If so, a warning is logged (`ŌÜĀ’ĖÅ Box broken ŌĆö SL should trigger`) ŌĆö the actual exit is handled by the Bitget SL order.

## 4. state.json Effects

When S1 evaluates a pair, the following `pair_states[symbol]` fields are updated:

```python
{
  "rsi":         float,    # current 3m RSI value
  "adx":         float,    # current daily ADX value
  "rsi_ok":      bool,     # whether RSI is in threshold zone
  "trend_ok":    bool,     # whether daily ADX + EMA filter passes
  "htf_bull":    bool,     # 1H higher high detected
  "htf_bear":    bool,     # 1H lower low detected
  "consolidating": bool,   # whether 3m coil is detected
  "box_high":    float,    # 3m coil box upper boundary
  "box_low":     float,    # 3m coil box lower boundary
  "s1_signal":   str,      # "LONG" | "SHORT" | "HOLD"
  "reason":      str,      # S1 reason/debug string
  "strategy":    str,      # "S1" if S1 is the leading signal
  "signal":      str,      # aggregate signal (first non-HOLD across S1ŌĆōS6)
}
```

When an S1 trade is active, `open_trades` contains the trade dict with `strategy="S1"`.  
On close, the trade moves to `trade_history` (runtime list; CSV is authoritative for persistence).

## 6. Active Trade Behavior (per tick)

While an S1 trade is open, each scan tick:

1. **PnL sync**: `unrealised_pnl` updated from exchange position data.
2. **Partial TP detection**: if live position qty dropped below 75% of initial qty and `partial_logged=False`, a `S1_PARTIAL` row is written to CSV.
3. **Swing trail** (`S1_USE_SWING_TRAIL = True`): reference-high/low gated ŌĆö SL steps only after price breaks a structural 3m swing high (LONG) or swing low (SHORT) AND a new opposing swing forms after it. See ┬¦3 for full cycle.
4. **Box break warning**: advisory check on last closed 3m candle ŌĆö logs warning if box is broken.
5. **Close detection**: if position disappears from exchange, `S1_CLOSE` is logged to CSV and trade is removed from `active_positions`.

## 8. Trade History on Exit

On close, `S1_CLOSE` is appended to `trades.csv` with:
- `result`: `"WIN"` or `"LOSS"` (based on total PnL including partial)
- `pnl`: close portion PnL (partial PnL already separately logged)
- `exit_reason`: reason string (from paper_trader or Bitget realized PnL path)
- `exit_price`: mark price at close

Win/loss counter is incremented in state; if 3 losses on the same pair today, the pair is paused until UTC midnight.
